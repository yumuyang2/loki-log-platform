## 项目简介
基于三台阿里云 ECS（2核4G）构建的轻量级日志平台，使用 Loki + Promtail + Grafana 实现日志的集中采集、存储和可视化。通过容器化部署和 Ansible 自动化，大幅提升故障定位效率，并与现有 Prometheus 监控体系形成“指标+日志”一体化可观测性方案。

## 背景与挑战
- 电商网站原有日志分散在三台服务器，排查问题时需分别登录服务器 grep 日志，平均耗时 20 分钟。
- 缺乏日志集中存储和查询能力，无法关联分析。
- 手工部署效率低，难以保证多节点配置一致性。

## 技术栈
- **日志存储**：Grafana Loki（轻量级、低成本）
- **日志采集**：Promtail（支持容器日志自动发现）
- **可视化**：Grafana（集成现有监控面板）
- **容器化**：Docker（所有组件容器化运行）
- **自动化**：Ansible（配置管理、模板渲染、一键部署）

## 架构设计
- 三台 ECS 节点：Node1、Node2（应用服务器）、Node3（Loki 存储 + Grafana）
- 每台节点运行 Promtail 容器，采集系统日志、Docker 容器日志及 Nginx 访问日志
- 所有日志发送至 Node3 的 Loki 服务，通过 Grafana 统一查询
